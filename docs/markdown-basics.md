# 📝 Markdown Basics

Markdown is a lightweight markup language with plain text formatting syntax. It is widely used for writing project READMEs, creating wiki pages, and documenting code in collaborative git repositories. 💻

---

## 🏗️ Formatting and Layout elements

### 1️⃣ Headers Hierarchy
Organize sections logically using `#` symbols:
```markdown
# Header 1 (Title)
## Header 2 (Major Sections)
### Header 3 (Sub-sections)
```

### 2️⃣ Text Styling
Style text elements clearly:
* **Bold:** `**text**` -> **text**
* **Italic:** `*text*` -> *text*
* **Strikethrough:** `~~text~~` -> ~~text~~
* **Code Inline:** `` `code` `` -> `code`

### 3️⃣ Lists and Checklists
* **Unordered List:**
  ```markdown
  - Sapphire
  - Cobalt
  ```
* **Task Checklist:**
  ```markdown
  - [x] Onboarded
  - [ ] Star the repository
  ```

### 4️⃣ GFM Tables
Organize structured data cleanly:
```markdown
| Name | Role | Location |
| :--- | :--- | :--- |
| Shuvranshu | Lead | India |
| Alice | Developer | USA |
```

### 5️⃣ GitHub Alerts (Admonitions)
Create warning or information boxes:
```markdown
> [!NOTE]
> Helpful background information.

> [!WARNING]
> Critical precautions to keep in mind.
```
